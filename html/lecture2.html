Gradient Descent

Manually finding the best weights is tedious. We can use Gradient Descent to automate this.

Imagine you are on a mountain (the loss function) and want to get to the bottom (minimum loss). You look around to see which direction goes down the steepest, and you take a step in that direction. You repeat this until you reach the bottom.

Mathematically, the "direction" is the negative of the gradient (derivative) of the loss with respect to the weights.

learning_rate = 0.01
epochs = 1000

plt.scatter(x, y)
for i in range(epochs):
    # 1. Forward pass: Compute predicted y
    y_pred = a * x**2 + b * x + c
    
    # 2. Compute loss
    loss = mse(y, y_pred)
    
    # 3. Backward pass: Compute gradients
    loss.backward()
    
    # 4. Update weights
    # We use torch.no_grad() because we don't want to track this operation in the gradient graph
    with torch.no_grad():
        a -= learning_rate * a.grad
        b -= learning_rate * b.grad
        c -= learning_rate * c.grad
        
        # Zero the gradients after updating
        a.grad.zero_()
        b.grad.zero_()
        c.grad.zero_()
    
    if i % 100 == 0:
        print(f"Epoch {i}: Loss = {loss.item():.4f}, a = {a.item():.2f}, b = {b.item():.2f}, c = {c.item():.2f}")
        plt.plot(x, y_pred.detach().numpy(), alpha=(i+100)/epochs, color='blue')
plt.show()
